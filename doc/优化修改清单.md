### 1. 单例模式重构：DCL → Meyer's Singleton

**涉及文件**：[logger_p.h](..\logger\private\logger_p.h#L217-L219) / [logger_p.cpp](..\logger\private\logger_p.cpp#L82-L83) / [logger_p.cpp:432-446](..\logger\private\logger_p.cpp#L432-L446)

**问题**：
- `s_mutex` + `s_instance` + DCL（双检锁），共需 3 个静态变量 + ~10 行代码
- `s_instance` 是原始指针（非 `std::atomic`），DCL 在 C++11+ 非原子模型下存在潜在重排风险
- 析构函数为 `= default`，单例 `new` 出来但从无 `delete`，日志缓冲可能未 flush

**方案**：用 C++11 保证线程安全的局部静态变量替代。

**修改前** (`logger_p.h` line 216-219)：
```cpp
private:
    static std::mutex s_mutex;
    static LogPrivate* s_instance;
```

**修改后**：
```cpp
// 删除 s_mutex 和 s_instance 声明
```

**修改前** (`logger_p.cpp` line 82-83)：
```cpp
LogPrivate* LogPrivate::s_instance = nullptr;
std::mutex LogPrivate::s_mutex;
```

**修改后**：
```cpp
// 删除这两行
```

**修改前** (`logger_p.cpp` line 432-446)：
```cpp
LogPrivate* LogPrivate::getInstance()
{
    if (s_instance == nullptr)
    {
        std::lock_guard<std::mutex> lock(s_mutex);
        if (s_instance == nullptr)
        {
            s_instance = new LogPrivate();
        }
    }
    return s_instance;
}
```

**修改后**：
```cpp
LogPrivate& LogPrivate::getInstance()
{
    static LogPrivate instance;  // C++11 保证线程安全
    return instance;
}
```

**联动修改**：所有调用 `getInstance()->xxx` 改为 `getInstance().xxx`（涉及 `logger_p.cpp` 约 30 处）。

**影响范围**：
- `logger_p.h`: 删除 `s_mutex`、`s_instance` 声明；`getInstance()` 返回类型改 `LogPrivate*` → `LogPrivate&`
- `logger_p.cpp`: 删除静态变量定义；重写 `getInstance()`；所有 `getInstance()->` → `getInstance().`

**工作量**：~30 分钟
**风险**：低（行为完全等价）

### 2. 六个日志级别方法去重

**涉及文件**：[logger_p.cpp:161-322](..\logger\private\logger_p.cpp#L161-L322) / [logger_p.h:54-107](..\logger\private\logger_p.h#L54-L107)

**问题**：`trace`/`debug`/`info`/`warn`/`error`/`critical` 六个方法各 ~20 行，逻辑完全相同，仅 spdlog 调用方法和 `m_xxxShowLine` 不同。

**方案**：抽取一个私有模板方法 `logImpl`，六个方法变为一行委托。

**修改后新增** (`logger_p.h` private 区域)：
```cpp
template<typename F>
static void logImpl(const char* fileName, int fileLine, const char* function,
                    const std::initializer_list<std::any>& msgList,
                    bool showLine, F&& spdlogFunc);
```

**修改后** (`logger_p.cpp`，六个方法之一示例)：
```cpp
void LogPrivate::trace(const char* fileName, int fileLine, const char* function,
                       const std::initializer_list<std::any>& msgList)
{
    logImpl(fileName, fileLine, function, msgList, m_traceShowLine,
        [this](auto&&... args) { getInstance().getLogger()->trace(std::forward<decltype(args)>(args)...); });
}
```

**模板实现** (`logger_p.cpp`)：
```cpp
template<typename F>
void LogPrivate::logImpl(const char* fileName, int fileLine, const char* function,
                         const std::initializer_list<std::any>& msgList,
                         bool showLine, F&& spdlogFunc)
{
    std::string msg = linkString(fileName, fileLine, function, msgList);
    if (msgList.size() == 1 && msg.empty()) {
        spdlogFunc("[{}:{}][{}] 日志内容为空", fileName, fileLine, function);
        return;
    }
    if (msgList.size() > 1 && msg.empty()) {
        spdlogFunc("[{}:{}][{}] 日志打印失败，数据类型转换错误", fileName, fileLine, function);
        return;
    }
    if (!showLine) { spdlogFunc("{}", msg); return; }
    spdlogFunc("[{}:{}][{}]{}", fileName, fileLine, function, msg);
}
```

**收益**：`logger_p.cpp` 减少 ~150 行，修改日志格式只需改一处。

**工作量**：~40 分钟
**风险**：低（纯重构）

### 3. `anytostring.hpp` 类型分发改为 O(1) 查找

**涉及文件**：[anytostring.hpp:45-185](..\logger\include\logger\anytostring.hpp#L45-L185)

**问题**：14 个 `if (data.type() == typeid(X))` 串联比较，最坏 O(n)。每次日志调用都会经过此路径。

**方案**：使用 `std::unordered_map<std::type_index, std::function<...>>` 实现 O(1) 分发。

**修改后**：
```cpp
#include <typeindex>
#include <unordered_map>

static const auto& getConverters()
{
    static const std::unordered_map<std::type_index,
        std::function<std::optional<std::string>(const std::any&)>> map = {
        { typeid(std::string),       [](auto& a) { return std::any_cast<std::string>(a); } },
        { typeid(const char*),       [](auto& a) { return std::any_cast<const char*>(a); } },
        { typeid(char*),             [](auto& a) { return std::any_cast<char*>(a); } },
        { typeid(int),               [](auto& a) { return std::to_string(std::any_cast<int>(a)); } },
        { typeid(unsigned int),      [](auto& a) { return std::to_string(std::any_cast<unsigned int>(a)); } },
        { typeid(long),              [](auto& a) { return std::to_string(std::any_cast<long>(a)); } },
        { typeid(long long),         [](auto& a) { return std::to_string(std::any_cast<long long>(a)); } },
        { typeid(float),             [](auto& a) { return std::to_string(std::any_cast<float>(a)); } },
        { typeid(double),            [](auto& a) { return floatingNumToString(std::any_cast<double>(a)); } },
        { typeid(size_t),            [](auto& a) { return std::to_string(std::any_cast<size_t>(a)); } },
        { typeid(uint64_t),          [](auto& a) { return std::to_string(std::any_cast<uint64_t>(a)); } },
        { typeid(bool),              [](auto& a) { return std::any_cast<bool>(a) ? "true" : "false"; } },
        { typeid(std::wstring),      [](auto& a) { /* UTF-16→UTF-8 转换 */ } },
        { typeid(wchar_t*),          [](auto& a) { /* 同上 */ } },
        { typeid(const wchar_t*),    [](auto& a) { /* 同上 */ } },
    };
    return map;
}

inline std::optional<std::string> anyToString(const char*, int, const char*, const std::any& data)
{
    if (!data.has_value() || data.type() == typeid(nullptr))
        return std::nullopt;

    auto& converters = getConverters();
    auto it = converters.find(std::type_index(data.type()));
    if (it != converters.end())
        return it->second(data);
    return std::nullopt;
}
```

**收益**：O(n)→O(1)，扩展新类型只需加一行 map 条目。

**工作量**：~30 分钟
**风险**：低（注意 `std::string&` / `const std::string&` 等引用类型的 `typeid` 匹配需验证）

### 4. `<codecvt>` 废弃警告处理

**涉及文件**：[anytostring.hpp:12](..\logger\include\logger\anytostring.hpp#L12) / [logger_p.cpp:18-21](..\logger\private\logger_p.cpp#L18-L21)

**问题**：`std::wstring_convert<std::codecvt_utf8_utf16<wchar_t>>` 在 C++17 已标记 `[[deprecated]]`，C++26 将彻底移除。

**方案**：条件编译，Windows 用 `WideCharToMultiByte`，POSIX 用 ICU 或自定义转换。

**修改后** (`anytostring.hpp`)：
```cpp
#ifdef _WIN32
std::string toUtf8(std::wstring_view wstr) {
    if (wstr.empty()) return {};
    int len = WideCharToMultiByte(CP_UTF8, 0, wstr.data(), (int)wstr.size(), nullptr, 0, nullptr, nullptr);
    std::string result(len, '\0');
    WideCharToMultiByte(CP_UTF8, 0, wstr.data(), (int)wstr.size(), result.data(), len, nullptr, nullptr);
    return result;
}
#else
// std::wstring_convert 暂时保留（POSIX 侧移除较晚）
#endif
```

**工作量**：~20 分钟
**风险**：中（需在目标平台验证 UTF-8 转换正确性）

### 5. `floatingNumToString` 用 `std::to_chars` 重写

**涉及文件**：[anytostring.hpp:18-43](..\logger\include\logger\anytostring.hpp#L18-L43)

**问题**：循环中做 double→string→double 往返转换，对一个数最多转换 8 次。

**方案**：用 C++17 的 `std::to_chars` 一次完成。

**修改后**：
```cpp
#include <charconv>

inline std::string floatingNumToString(double value)
{
    if (value == 0.0 || value == -0.0)
        return "0";
    char buf[32];
    auto [ptr, ec] = std::to_chars(buf, buf + sizeof(buf), value);
    if (ec != std::errc{})
        return std::to_string(value);  // fallback
    return std::string(buf, ptr);
}
```

**注意**：MinGW 的 `std::to_chars` 对浮点数支持可能不完整（GCC < 11），需验证或加 `#ifdef` fallback。

**工作量**：~15 分钟
**风险**：中（依赖编译器支持，需在 MinGW 下验证）

### 6. `loadDefaultConfig` 结构化解耦

**涉及文件**：[logger_p.cpp:675-843](..\logger\private\logger_p.cpp#L675-L843)

**问题**：~170 行函数，每个 sink 类型单独声明变量 → 设值 → 推入节点，模式高度重复。

**方案**：定义 `SinkDefaults` 结构体 + 数组 + 循环。

**修改后思路**：
```cpp
struct SinkDefaults {
    std::string type;
    std::string level;
    std::string file_path;
    // ... 其他可选参数
    std::function<void(YamlTool::YamlNode&)> setExtra;  // 自定义额外字段
};

static const SinkDefaults defaultSinks[] = {
    { SINK_TYPE_STDOUT_COLOR_SINK_MT, "trace" },
    { SINK_TYPE_BASIC_FILE_SINK_MT, "trace", "./logs/basic_file_sink_mt.log", {},
      [](auto& n) { YamlTool::YamlTool::setDef<std::string>(n, "truncate", "false"); } },
    // ... 6 个 sink 声明为数据，不是代码
};

// 循环处理
for (const auto& s : defaultSinks) {
    YamlTool::YamlNode node;
    YamlTool::YamlTool::setDef(node, "type", s.type);
    YamlTool::YamlTool::setDef(node, "level", s.level);
    // ... 公共字段
    if (s.setExtra) s.setExtra(node);
    YamlTool::YamlTool::pushBack(sinksNode, node);
}
```

**收益**：~170 行 → ~60 行，新增默认 sink 只需加一行数据。

**工作量**：~60 分钟
**风险**：中（YAML 结构需与现有输出一致）

### 7. `count_rotating_file_mt` 重复边界检查合并

**涉及文件**：[count_rotating_file_mt_sink.hpp:94-118](..\logger\private\count_rotating_file_mt_sink.hpp#L94-L118)

**问题**：`log_count_ >= max_count_` 检查在 `sink_it_` 中出现两次，中间夹着 `ensure_opened_for_write_`。

**方案**：将滚动检查集中在 `ensure_opened_for_write_` 之前一次完成：

```cpp
void sink_it_(...) override
{
    if (!rotated_on_open_done_) { /* 保持 */ }

    ensure_opened_for_write_();  // 内部处理 max_count 检查和滚动

    // 写入（此时保证 log_count_ < max_count_）
```

让 `ensure_opened_for_write_` 内部一次性处理「打开→检查→必要时滚动→再打开」，避免 `sink_it_` 中的重复判断。

**工作量**：~30 分钟
**风险**：中（需仔细验证边界条件：首次写入、max_count=0/1 等）

### 13. `ID8Generator` 每次 `operator()` 都读时钟

**涉及文件**：[id8generator.hpp:77-79](..\logger\private\id8generator.hpp#L77-L79)

**问题**：每次生成 ID 都调用 `steady_clock::now()`，对于高频场景（如批量添加 callback sink）不必要的系统调用。

**方案**：clock 仅在初始化时读取，operator() 只用原子计数器 + mix：

```cpp
std::string operator()() noexcept
{
    uint64_t c = counter_.fetch_add(1, std::memory_order_relaxed);
    uint64_t val = mix64(c) % MAXVAL;
    // ... 转为 8 字符
}
```

种子的唯一性由构造时混入的 PID + clock + random_device 保证，后续 ID 只靠计数器即可。

**工作量**：~5 分钟
**风险**：低
